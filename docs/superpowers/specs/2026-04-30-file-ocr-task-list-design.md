# 文件OCR任务列表与新增流程设计（直改版）

## 1. 背景与目标

当前 `文件OCR > 任务列表` 页面仅有统计与占位内容，尚未提供完整任务管理能力。  
本次按“前后端一体直改（最快）”落地以下能力：

- 任务列表查询与条件过滤；
- 新增文件OCR两屏流程（选择OCR类型、上传文件）；
- 上传到S3后创建任务记录；
- 页面内展示上传进度；
- 新增流程对话框全程不可关闭。

## 2. 范围与边界

### 2.1 本次范围

- 列表字段、筛选、分页。
- 新增任务两屏流程与提交逻辑。
- `ocr_file` 表字段调整（新增与删除）。
- 上传格式/大小/数量校验与进度展示。
- 操作列提供入口：查看详情、重新OCR、下载结果、删除任务、取消处理中任务。

### 2.2 非本次范围

- OCR解析引擎内部流程重构。
- 大规模任务编排系统或队列重构。
- 与本次目标无关的页面/模块改造。

## 3. 需求确认结果（冻结）

### 3.1 列表展示字段

- 文件名
- OCR类型
- 状态
- 文件大小
- 文件路径（展示 `object_key`）
- 页数
- 创建时间
- 更新时间
- 操作

### 3.2 列表筛选条件

- `file_name`
- `ocr_type`
- `status`
- 创建时间范围（`create_at`）

### 3.3 分页规则

- 默认每页 `10` 条。
- 提供页大小下拉框，允许用户切换每页条数。

### 3.4 新增任务流程

- 两屏流程：
  - 第一屏：选择 `PaddleOCR` 或 `MinerU`；
  - 第二屏：上传文件；本屏点击“完成”触发上传与入库。
- 每屏有“上一步/下一步”流程控制；第二屏使用“上一步/完成”。

### 3.5 上传规则

- 仅允许：`pdf`、`jpg`、`jpeg`、`png`。
- 不允许：`gif` 与其他格式。
- 单文件大小限制：`50MB`。
- 单次最多文件数：`50`。
- 文件选择组件使用手动上传模式，不自动上传。
- 只有用户点击上传/完成后，才实际调用S3上传并创建记录。

### 3.6 对话框行为

新增流程对话框在流程期间不可关闭：

- 禁止点遮罩关闭；
- 禁止 ESC 关闭；
- 禁止右上角关闭按钮；
- 上传执行中不提供关闭路径。

### 3.7 状态与字段更新

- 新建记录初始状态：`INIT`（已创建待处理）。
- 页数默认空（`NULL`），不在创建时写入。
- 解析完成后，统一更新：`page_count`、`status`、`update_at`。

### 3.8 数据字段变更

`ocr_file`：

- 新增：`file_size`（文件大小）、`object_key`（文件存储对象）。
- 删除：`file_uri`。

## 4. 数据模型与数据库设计

## 4.1 表结构调整（`ocr_file`）

新增字段建议：

- `file_size BIGINT NULL`：存储文件大小（字节）。
- `object_key VARCHAR(1024) NOT NULL`：S3对象键，作为文件路径展示字段。
- `page_count INTEGER NULL`：页数，解析完成后回填。

删除字段：

- `file_uri`。

说明：

- `status` 继续复用现有状态体系（至少包含 `INIT`、`PROCESS`、`SUCCESS`、`FAILED`）。
- `create_at` 在创建时写入，`update_at` 在后续状态推进时更新。

## 5. 后端接口设计（直改）

## 5.1 列表查询接口

建议路径：`GET /workspaces/{workspace_id}/ocr-files`

查询参数：

- `file_name`（可选，模糊）
- `ocr_type`（可选，精确）
- `status`（可选，精确）
- `create_at_start`（可选）
- `create_at_end`（可选）
- `page`（默认1）
- `page_size`（默认10）

返回项（单行）：

- `id`
- `file_name`
- `ocr_type`
- `status`
- `file_size`
- `object_key`
- `page_count`
- `create_at`
- `update_at`

返回体包含分页总量信息，支持前端页码器和页大小切换。

## 5.2 新建任务接口

建议路径：`POST /workspaces/{workspace_id}/ocr-files`

请求体（一次提交多个文件）：

- `ocr_type`
- `files`（数组）
  - `file_name`
  - `file_size`
  - `object_key`

处理流程：

1. 校验 `ocr_type` 与文件数量。
2. 校验文件扩展名与大小约束。
3. 为每个文件创建 `ocr_file` 记录：
   - `status=INIT`
   - `page_count=NULL`
4. 返回成功/失败明细，便于前端提示。

说明：S3上传由前端在“完成”触发后调用现有上传接口，后端创建接口仅接收上传成功后的元数据并入库。

## 5.3 操作列能力对接策略

操作入口统一放在列表“操作”列：

- 查看详情
- 重新OCR（保留原文件重跑）
- 下载结果
- 删除任务
- 取消处理中任务

本期以“可落地、不中断主流程”为原则：

- 已具备后端能力的动作直接联通；
- 未具备后端能力的动作先返回明确提示，不阻塞“查询+新增”主流程交付。

## 6. 前端交互与页面设计（直改）

## 6.1 任务列表页

- 顶部筛选区：`file_name`、`ocr_type`、`status`、创建时间范围。
- 列表按 `create_at desc` 默认排序。
- 分页默认 `10`，页大小可切换。
- 列表刷新触发时机：
  - 查询/重置；
  - 新建任务成功后；
  - 操作列动作完成后（若影响列表）。

## 6.2 两屏新增弹窗

第一屏（OCR类型）：

- 单选 `PaddleOCR` / `MinerU`。
- “下一步”仅在已选择时可用。

第二屏（文件上传）：

- 组件为手动上传模式。
- 文件校验在选择时执行，非法文件直接拦截并提示。
- 按钮：
  - 上一步：返回第一屏并保留已选信息。
  - 完成：触发上传 + 入库创建。

## 6.3 上传进度与结果反馈

- 第二屏展示每个文件的上传进度条。
- “完成”后进入提交态，防止重复点击。
- 对每个文件展示结果（成功/失败及原因）。
- 全部完成后刷新列表，并在页面提示整体结果摘要。

## 7. 关键时序（简化）

1. 用户打开“新增文件OCR”。
2. 第一屏选择 `ocr_type`，点击下一步。
3. 第二屏选择文件（手动模式，不自动上传）。
4. 用户点击完成。
5. 前端逐个/并发调用现有S3上传接口，获取 `object_key`。
6. 前端调用新建任务接口写入 `ocr_file`（`status=INIT`）。
7. 前端展示结果并刷新任务列表。

## 8. 错误处理

- 文件格式非法：阻止加入上传队列并提示。
- 文件超限：阻止加入上传队列并提示。
- 文件数量超限：阻止继续选择并提示。
- S3上传失败：该文件标记失败，不影响其他文件提交。
- 入库失败：返回失败原因并在前端逐条提示。
- 网络异常：保留当前弹窗状态，允许用户重试当前步骤。

## 9. 测试要点

### 9.1 后端

- 列表过滤组合查询正确性（单条件/多条件/空条件）。
- 分页正确性（默认10、切换页大小）。
- 新建接口对字段入库正确性（`file_size/object_key/status/page_count`）。
- 删除 `file_uri` 后的模型与返回一致性。

### 9.2 前端

- 两屏流程按钮状态正确（下一步可用条件、上一步回退保留）。
- 上传手动触发，不会选中文件后自动上传。
- 进度条按文件更新。
- 对话框不可关闭约束生效（遮罩/ESC/右上角）。
- 上传成功后列表刷新；失败提示清晰。

### 9.3 联调

- 现有S3上传接口返回 `object_key` 的联调闭环。
- 批量上传（边界：1、50）行为验证。
- 大文件边界（50MB）校验验证。

## 10. 实施顺序（直改）

1. 后端：`ocr_file` 模型与SQL字段调整。
2. 后端：列表查询接口、创建接口。
3. 前端：列表筛选+表格列+分页。
4. 前端：两屏弹窗与手动上传、进度条、不可关闭约束。
5. 前后端联调与回归验证。

## 11. 验收标准

- 任务列表支持按 `file_name/ocr_type/status/创建时间范围` 查询。
- 列表展示字段与产品确认一致。
- 新增流程为两屏，第二屏点击“完成”才上传并入库。
- 文件限制（格式、大小、数量）全部生效。
- 对话框流程中不可关闭。
- 新建任务初始为 `INIT`，`page_count` 初始为空。
- `file_uri` 已移除，列表路径展示 `object_key`。
